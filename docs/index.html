02 // Pipeline

Four Agents. One Command.

Each stage has a distinct responsibility. Scientific content is invariant across all four stages — results, equations, citations are never altered.

STAGE 01
Author
Senior researcher persona. Writes a complete technical draft from raw input: paragraph structure, scientific verbs, one clear function per paragraph. Covers the full paper lifecycle from problem definition to conclusions.
STAGE 02
Editorial
Linguistics specialist. Improves sentence rhythm, syntactic variety, connector logic, and redundancy removal. Does not touch scientific content.
STAGE 03
IEEE Style
Enforces IEEE Transactions and IEEE Access conventions: impersonality, acronym protocol, terminological consistency, voice balance, forbidden constructions, precision vocabulary.
STAGE 04
Reviewer
Anonymous peer reviewer. Identifies unsupported claims, coherence failures, reproducibility gaps, and structural issues. Fixes what can be fixed; flags the rest in a structured Reviewer Report.
03 // Properties

Built for Peer Review

🔒
Scientific Preservation
Results, hypotheses, equations, symbols, numerical values, units, citations, and citation order are invariant across all four pipeline stages. Only linguistic form is subject to transformation.
📄
LaTeX Native
Reads .tex files directly. Preserves all LaTeX markup, environments, macros, and bibliography commands. Only prose text passes through the pipeline.
🌐
Multi-Journal
IEEE Transactions, IEEE Access, Elsevier, Springer, ACM, Nature Portfolio, and Qualis A (CAPES). Input in Portuguese or English — output always in formal academic English.
🤝
30+ Agents Supported
One installer detects Claude Code, Codex, Gemini, Cursor, Windsurf, Cline, GitHub Copilot, opencode, OpenClaw, and 20+ others. Installs the right mechanism for each agent automatically.
